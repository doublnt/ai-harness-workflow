# Expert Review

Expert roles must be project-specific.

## Role schema

Each expert role must include:

- name
- purpose
- appliesTo paths or concepts
- required context
- project invariants
- blocker criteria
- evidence requirements
- output format

## Review output

```text
Expert Role:
Scope:
Evidence Read:
Blockers:
Needs Changes:
Non-blocking Suggestions:
Unknowns:
Required Evidence:
Verdict: Pass / Needs Changes / Blocked
```

## Important

Do not use "as a security expert" as a substitute for a rubric. Every role must have concrete blocker criteria.
