# Template Contract

Generated governance files must be:

1. concrete, not generic
2. reviewable
3. target-specific
4. safe by default
5. non-overwriting
6. explicit about unknowns

## Required concepts

Every target must include:

- project context
- engineering constitution
- risk levels
- file change policy
- workflow overview
- requirement gate
- design gate
- implementation gate
- code review gate
- testing gate
- security gate
- release gate
- templates for feature spec, design proposal, ADR, implementation plan, test plan, PR review, release plan

## Forbidden output

Targets must not create `.ai/` directories.
